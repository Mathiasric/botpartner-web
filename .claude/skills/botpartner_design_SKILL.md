This skill guides design improvements for Botpartner.no within strict technical and strategic constraints.

## Context

Botpartner.no is a high-ticket B2B lead generation system.

Every design decision must improve:
- Clarity
- Trust
- Authority
- Conversion

Design philosophy: **Premium minimal technical**

Aesthetic references: Stripe, Linear, Vercel

## Technical Constraints (Non-Negotiable)

- Static HTML only
- Vanilla CSS only (single file: botpartner_web.css)
- Vanilla JavaScript only (minimal: navigation, scroll, mobile menu)
- No frameworks
- No build tools
- No dependencies

## Design Principles

### What to Improve

**Typography**
- Use refined, professional fonts (not generic system fonts)
- Clear hierarchy: H1 → H2 → H3
- Generous line-height for readability
- Consider: Inter Display, Space Grotesk, Departure Mono, Berkeley Mono, Söhne
- Avoid: Arial, Helvetica, generic system fonts

**Color & Contrast**
- Maintain clean, minimal palette
- Use CSS variables for consistency
- Improve contrast for readability
- Subtle accent colors for CTAs
- Professional gradients (linear, refined, not "startup purple")

**Spacing & Layout**
- Generous white space
- Consistent vertical rhythm
- Aligned grid structure
- Clear visual hierarchy
- Breathing room around CTAs

**Subtle Motion** (allowed, but restrained)
- Smooth transitions on hover (0.2s - 0.3s)
- Fade-in on scroll (intersection observer + CSS)
- Micro-interactions on buttons
- No aggressive animations
- No parallax experiments

**Background & Texture**
- Subtle gradients
- Light grain/noise for depth
- Clean geometric patterns (if needed)
- Professional, not decorative

### What to Avoid

❌ Bold maximalism
❌ Aggressive animations
❌ Complex layouts
❌ Asymmetric chaos
❌ Decorative illustrations
❌ Overly colorful startup aesthetics
❌ Generic "AI startup" purple gradients
❌ Heavy visual experiments

## Execution Model

When improving design:

1. **Verify first**: Read existing HTML/CSS structure
2. **Minimal scope**: Only modify what's necessary
3. **Test on mobile**: Preserve responsive behavior
4. **Maintain CTAs**: Don't break conversion flow
5. **Return diffs only**: No full file dumps

## Design Decisions Framework

Before implementing, ask:

- Does this improve clarity?
- Does this strengthen authority?
- Does this improve conversion likelihood?
- Is this aligned with Stripe/Linear/Vercel aesthetic?
- Can this be implemented in vanilla CSS/JS?

If any answer is "no", reconsider the change.

## Common Improvements

**Cards**
- Add subtle shadow (not heavy drop-shadow)
- Smooth hover transitions
- Clean borders or subtle backgrounds
- Consistent padding and spacing

**CTAs**
- Clear visual hierarchy
- Hover states with smooth transitions
- Strong contrast
- Generous click targets

**Sections**
- Clear visual separation
- Consistent padding
- Subtle background variations
- Readable text hierarchy

**Navigation**
- Clean, minimal
- Smooth scroll behavior
- Clear active states
- Mobile-friendly

## Remember

Speed and clarity matter more than visual novelty.

Design serves conversion, not aesthetics for aesthetics' sake.

Every pixel should earn its place by improving trust, clarity, or action.
